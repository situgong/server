Refactor the LinguaSpark server (`d:\tools\server`) to run natively on Windows by replacing the C++-based translation engine with the WASM-based implementation from `d:\tools\MTranServer`.

## 1. Analysis & Preparation
- **Objective**: Remove hard-to-build C++ dependencies (`linguaspark`, `bergamot-translator` native bindings) and use `bergamot-translator.wasm` executed via a Rust WASM runtime.
- **Reference**: `d:\tools\MTranServer` uses `bergamot-translator.wasm` loaded via Node.js.
- **Challenge**: The `bergamot-translator.wasm` is likely compiled with Emscripten. Running this in Rust requires a runtime that supports Emscripten ABI or a compatible WASI environment. We will use `wasmtime` or `wasmer` (depending on Emscripten support).

## 2. Dependency Updates (`Cargo.toml`)
- **Remove**:
  - `linguaspark` (Git dependency, source of C++ bindings).
  - `bergamot-translator` (Native binding version).
- **Add**:
  - `wasmtime` (Runtime for executing the WASM module).
  - `wasi-common` (WASI context support).
  - `wasmtime-wasi` (WASI implementation).

## 3. Asset Migration
- Create a `wasm/` directory in `d:\tools\server`.
- Copy `d:\tools\MTranServer/src/lib/bergamot/bergamot-translator.wasm` to `d:\tools\server/wasm/`.
- Ensure model files (`.bin`, `.spm`) are available in `d:\tools\server/models/` (copy from `MTranServer` config path if needed).

## 4. Implementation (`src/translation.rs` & `src/wasm_engine.rs`)
- **Create `src/wasm_engine.rs`**:
  - Implement a `WasmEngine` struct that initializes `wasmtime::Engine` and `Linker`.
  - Load the `bergamot-translator.wasm` file.
  - Implement necessary host functions (imports) required by the WASM module (e.g., memory management, logging).
- **Update `src/translation.rs`**:
  - Re-implement `Translator` to use `WasmEngine`.
  - Implement `translate(text, from, to)`:
    - Write input text to WASM memory.
    - Call the underlying WASM translation function.
    - Read the result from WASM memory.
- **Update `src/main.rs`**:
  - Replace `linguaspark::Translator` initialization with the new local `Translator`.

## 5. Verification
- Build the project on Windows (`cargo build`).
- Run the server (`cargo run`).
- Test the `/translate` endpoint with a sample request.
